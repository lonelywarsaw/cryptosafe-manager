# CryptoSafe Manager

Менеджер паролей с шифрованной базой данных и графическим интерфейсом (PyQt6). Конфиденциальные данные шифруются на уровне приложения перед записью в SQLite.

---

## Видение проекта

**CryptoSafe Manager** — безопасное локальное хранилище паролей с мастер-паролем, шифрованием записей, журналом аудита и удобным окном управления. Проект разрабатывается по спринтам от базового каркаса до готового к распространению приложения.

Цели:
- Одна точка входа — мастер-пароль; ключ шифрования не хранится в открытом виде.
- Шифрование конфиденциальных полей (пароли, заметки) перед записью в БД.
- Разделение слоёв: интерфейс (View), логика и события (Controller), данные и шифрование (Model).
- Расширяемость: заглушки крипто и ключей заменяются на полноценную реализацию в следующих спринтах.

## Настройка и запуск

### 1. Клонирование и переход в каталог проекта

```bash
git clone <URL репозитория>
cd Crypto
```

(или откройте каталог проекта в IDE.)

### 2. Виртуальное окружение

В репозитории **нет** готового виртуального окружения — его нужно создать у себя на машине (папку venv обычно не коммитят в git).

**Windows (PowerShell или cmd):**

```bash
python -m venv .venv
.venv\Scripts\activate
```

**Linux / macOS:**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

После активации в начале строки появится `(.venv)`.

### 3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Запуск приложения

Из корня проекта:

```bash
python main.py
```

При первом запуске откроется мастер настройки: задайте мастер-пароль и выберите путь к файлу базы. Дальше можно добавлять, редактировать и удалять записи через меню «Правка», открывать настройки и журнал через «Вид».

### 5. Запуск тестов

```bash
python -m unittest discover -s tests -p "*test*.py" -v
```

С замером покрытия (нужен пакет `coverage`):

```bash
pip install coverage
coverage run -m unittest discover -s tests -p "*test*.py" -v
coverage report -m
```

## Архитектура (обзор MVC)

Поток данных и ответственность слоёв:

```
┌─────────────────────────────────────────────────────────────────────────┐
│  VIEW (Графический интерфейс)                                             │
│  src/gui/                                                                │
│  • main_window.py   — главное окно, меню, таблица записей                │
│  • setup_wizard.py  — мастер первой настройки (мастер-пароль, путь к БД) │
│  • settings_dialog  — настройки (тема, язык, таймауты)                   │
│  • theme.py, i18n.py, widgets/                                            │
└────────────────────────────────┬────────────────────────────────────────┘
                                 │
                    действия пользователя / отображение данных
                                 ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  CONTROLLER (Логика, состояние, события)                                 │
│  src/core/                                                               │
│  • config.py        — настройки приложения, путь к БД, загрузка/сохран.  │
│  • state_manager.py — блокировка сессии, буфер обмена, таймер неактивности│
│  • events.py        — шина событий (подписка/публикация)                  │
│  • key_manager.py   — вывод ключа из пароля и соли                        │
│  • crypto/          — интерфейс шифрования и заглушка (XOR)               │
└────────────────────────────────┬────────────────────────────────────────┘
                                 │
                    вызовы шифрования, запись/чтение, публикация событий
                                 ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  MODEL (Данные и хранилище)                                              │
│  src/database/                                                           │
│  • models.py — схема SQLite (vault_entries, audit_log, settings, …)      │
│  • db.py     — подключение к БД, курсор, init_schema, backup/restore     │
│  • vault.py  — CRUD записей, шифрование полей перед вставкой/обновлением  │
│  • audit.py  — подписка на события и запись в журнал аудита              │
└─────────────────────────────────────────────────────────────────────────┘
```

Кратко:
- **View** — только отображение и ввод; обращается к контроллеру и модели через вызовы из главного окна.
- **Controller** — конфиг, состояние сессии, события и ключи; не знает об окнах, только о данных и событиях.
- **Model** — схема БД, работа с SQLite и логика хранилища (vault, аудит); конфиденциальные поля шифруются перед записью.

## Что реализовано (Спринт 1)

- Первый запуск: мастер настройки, создание мастер-пароля и выбор пути к БД.
- Разблокировка по мастер-паролю.
- Хранилище: добавление, редактирование и удаление записей; пароли и заметки хранятся в зашифрованном виде (заглушка XOR, далее — полноценное шифрование).
- Настройки: тема, язык, таймаут буфера, авто-блокировка; сохранение в файл и в таблицу настроек при открытой БД.
- Журнал аудита и резервная копия файла БД.
- Тесты (unittest), в CI — проверка покрытия (≥70%).
